## 2. 코드 설계 (Pseudo Code)

```cpp
#include <iostream>
using namespace std;

int main (void) {
    ios_base::sync_with_stdio(false);cin.tie(0);cout.tie(0);
    int N = 0;
    cin >> N;
    int x[N] = {0};
    int y[N] = {0};
    for(int i = 0; i<N; i++) {
        cin >> x[i] >> y[i];
    }
    for(int j = 0; j<N; j++) {
        for(int i = 0; i<N-j-1; i++) {
            if(y[i]>y[i+1]) {
                int foo = y[i];
                y[i] = y[i+1];
                y[i+1] = y[i];

                foo = x[i];
                x[i] = x[i+1];
                x[i+1] = x[i];
            }
        }
        if(j>=1){
            if(y[j]==y[j-1]) {} // 아니 이렇게 하면 y가 막 3개 씩 같아 버리면 처리를 못하는데
        }
    }
    return 0;
}
```

## 3. 트러블 슈팅 (Retrospective)

### 내 코드의 문제점 (실패 이유)

- **Case:** 일단 정렬을 구현하려는 생각을 왜 했지?
- **Cause:** std::sort 사용, 이럴 땐 자체 비교 함수. **"a < b 가 참이면 a가 앞에 온다"**

### 해결 방안 (수정 사항)

- 비교 함수 추가
- pair 자료형 활용 (utility에 있음) .first .second 활용.

### 배운 점 (New Knowledge)

- sort는 매우 강력하다.
- sort에 함수 쓸 때 아니지 그냥 습관적으로 `const 자료형&` 활용하자.

#include <iostream>
#include <vector>
#include <utility>
#include <algorithm>
using namespace std;

int main (void) {
    ios_base::sync_with_stdio(false);cin.tie(0);cout.tie(0);
    int N = 0;
    cin >> N;
    vector<pair<int,int>> v(N);
    for(int i = 0; i<N; i++) {
        cin >> v[i].first >> v[i].second;
    }
    sort(v.begin(), v.end(), [](pair<int,int> a, pair<int,int> b){
        //[](const pair<int,int>& a, const pair<int,int>& b) 가 더 좋음.
        if(a.second!=b.second){
            return a.second < b.second;
        } else {
            return a.first < b.first;
        }
    });
    for(int i = 0; i<N; i++) {
        cout << v[i].first << " " << v[i].second << '\n';
    }
    return 0;
}
